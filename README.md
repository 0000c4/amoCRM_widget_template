## Использование amo-widget-builder

**amo-widget-builder** — это библиотека для сборки и отладки виджетов amoCRM.

### Разработка с «горячей» перезагрузкой (hot reload)

1. **Создание виджета-заглушки**
   Выполните команду:
   ```bash
   npm run widget:dev
   ```
   Будет собран «виджет-бэкдор» (`widget.zip`). Это заготовка, которая подключает код с локального сервера разработки.

2. **Запуск сервера разработки**
   Выполните команду:
   ```bash
   npm run serve
   ```
   Запустится веб-сервер Webpack Dev Server.
   - **Обязательно** откройте его URL в браузере (обычно `https://localhost:8080`).
   - Примите предупреждение системы безопасности о самоподписанном сертификате (нажмите «Продолжить» или «Дополнительно» → «Перейти на сайт»).

3. **Установка виджета в amoCRM**
   - Загрузите полученный на шаге 1 архив `widget.zip` в amoCRM как **приватную интеграцию**.
   - Виджет автоматически начнёт использовать код с локального сервера, позволяя instantly видеть изменения при сохранении файлов.

## Временный хотфикс для генерации плейсхолдеров картинок

В текущей версии библиотеки `amo-widget-builder` сломалась генерация плейсхолдеров для изображений. Для временного решения проблемы выполните следующее:

### Шаги для исправления:

1. **Откройте файл библиотеки:**
   ```
   /node_modules/amo-widget-builder/dist/widget-builder.js
   ```

2. **Найдите строку 89** (или область, где указан URL для плейсхолдеров изображений)

3. **Измените URL-адрес** на:
   ```javascript
   'https://fakeimg.ryd.tools/'
   ```

### Важное ограничение!
⚠️ **Данный URL не работает с IP-адресов из РФ**

### Альтернативные опции (если основной URL недоступен):

**Вариант 1:** Использовать другой сервис плейсхолдеров
**Вариант 2:** Использовать локальные плейсхолдеры
